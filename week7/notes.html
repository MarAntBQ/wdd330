Testing and Debugging



It’s really important to learn about it, because we need to minimize errors, and have a way to identify them.

In errors we can have system errors “system or external devices with which the program is interacting”, programmer errors “incorrect syntax or faulty logic”, and user errors “incorrect data, and the program can’t handle them.”

Exceptions are used by the program in order to deal with errors. Stack trace is proceeded in backwards with an exception.

Warnings occurs when there’re not errors, and the code won’t cause the program to crash, but it can be problematic, or it’ll run incorrectly.

Importance: JavaScript didn’t implement exceptions at all until ECMAScript 3, ninja programmer should ensure that the code they write fails loudly in development so any errors can be identified and fixed quickly.

Strict Mode: It produces more exceptions and warnings and prohibits the use of some deprecated features. It’s better to find the errors asap, than after in order to fix them. To implement we should put this code: 'use strict';
ES6 introduced JavaScript modules, as self-contained pieces of code that are in strict mode by default, the declaration isn’t required.

Linting Tools: It can be used to test the quality of code, beyond the strict mode. They enforce a programming style guide. Good solution for teams’ work.

Feature Detection: The recommended way to determine browser support for a feature is to use feature detection. The “old-school” way of checking for browser support was known as browser sniffing.

Debugging in the Browser: The process of finding out where bugs occur in the code and then dealing with them. It’s useful to use breakpoints, they help us to view the value of different variables at that point in the program.

The Trusty Alert: I love this way, In PHP I use echo, in NodeJS console.log(), but the alert stops a program from running until OK is clicked.

Using the Console: It provides a number of methods for logging information and debugging. This can be used to log the value of variables at different stages of the program, but it won’t stop.

Debugging Tools: Modern browsers also have breakpoints in debugging tool, so in other words, while running we can change variables at those points. Useful command is debugger keyword.

Error Objects: It can be created by the host environment when an exception occurs, or a constructor function, which can take a parameter, there are more error objects:

EvalError is not used in the currently and only retained for backwards compatibility.

RangeError is when a number is outside a range.

ReferenceError is when a reference is made to an item that doesn’t exist.

SyntaxError is when there’s a code’s syntax error.

TypeError is when there’s an error in the type of value used.

URIError is hen there’s a problem encoding.

InternalError is a non-standard error that is thrown when an error occurs in the JavaScript engine.


They have different properties: name “returns the name of the error constructor”, message “returns a description of the error” and stack “return a stack trace for that error”.

Throwing Exceptions: It’s also possible to throw your own exceptions using the throw statement. It can be applied to any JavaScript expression, causing the execution of the program to stop.

Exception Handling: When an exception occurs, the program terminates with an error message, it helps us to identify and fix errors. We can do it by catching the error.
If we suspect a piece of code will result in an exception, we can wrap it in a try block, but if an exception occurs it will pass the error object that is thrown onto a catch block.

Tests: It’s an important part of programming, it’s really important to make our best to write good tests, in order to identify the potential errors early. It can be a simple function, so we need to run and make different tests.

Test-driven Development: It’s a process of writing tests before any actual code. They’ll fail because there’s not code to test, after write the code to pass the test, after it should be refactored, continue test at each stage, and finally write more tests for new features.
It can be hard to always use, but it’s important to make tests

Testing Frameworks: Writing tests code can be laborious, frameworks provide us a good way to use, and there’re differents:

Jest created by Facebook: It makes it easy to create and run tests by providing helper methods for common test assertions.

Crunching Some Numbers: We can create some libraries in order to work with different functions, and it’s really cool how to work with this Jest framework, so we follow the steps until our library of functions is growing, we should continue with the process of refactoring. This demonstrates how TDD can be used to keep adding functionality in small increments using the fail, pass and refactor cycle.
