Object-oriented programming (OOP):


We learned about objects before so it’s this style that involves them with their properties and methods, it’s used to model representations of objects in the real world.

We have 3 concepts:

Encapsulation: Keep all the programming logic inside an object and making methods available to implement the functionality.

Polymorphism: Various objects can share the same method, but they can override shared methods with a different implementation.

Inheritance: Take an object that exists and inherit all its properties and methods, also we can improve on its functionality by adding new properties and methods.

Classes: To define a blueprint for an object with the properties and methods.

Constructor functions are the way that we model the object where we create the properties, and give the functions that it can perform.

We can use Object, Array and Function to create objects, arrays and functions.

With ES6 we have a new class declaration syntax “it makes sense,” and it’s more understandable than constructer function syntax, in my case I think that it’s easier to read.

Static methods are called directly by the class rather than by instances of the class.

Prototypal inheritance model means that every class has a different prototype property that is shared by every instance of the class.

Prototype property of the class are useful if we don’t have access to the class declaration, but we want to add properties and methods.

If we want to know the prototypes, we can use the constructor function’s prototype property or Object.getPrototypeof() method “it’s required to send the object as parameter”.

It’s really cool to know about the Prototype Is Live in case the “ninja turtles” in case that a new property of method is added so its class will inherit the new properties and methods automatically.

Methods are like functions that an object can perform so we have public and private ones. It’s really important that we can understand the differences and give access through public to what’s required for.

The constructors for objects are important: “When an object calls a method, the JavaScript engine will check to see if the object has that method. If it doesn’t, it will check if the object’s prototype has the method. If not, it will check whether the prototype’s prototype has it”

If properties are not enumerable, they won’t show up when a for-in loop is used to loop through an object’s properties and methods.

A class can inherit from another class using the extends keyword in a class declaration, it’s really curious because I used something like that with PHP in SQL DB connection, so I extended the connections in my models, so the purpose was to access all the properties of my connection.

We can override a method with Polymorphism. For example, a new way to present a string, etc. Monkey-Patching is adding more methods to the prototype of JavaScript’s built-in objects — such as Number, String, and Array — to add more functionality.

A property descriptor is where the Property Attributes and Descriptors are stored. The get() and set() methods can be used to control how a property is set using assignment and the value that is returned when a property is queried.

We can use the create method in order to create a new object “exact copy of the object that is provided as an argument.” Creating objects from objects will create a prototype chain.

Mixins: A way of adding properties and methods of some objects to another object without using inheritance. One use for the mixin() function is to add a large number of properties to an object all at once, for example it’s like creating an entire array:

Factory Functions: can be used to return an object and we can mix with other methods for example copy() and mixin().

If a method returns this, its methods can be chained together to form a sequence of method calls that are called one after the other.

A common solution is to set the variable that to equal this before the nested function, and refer to that in the nested function instead of this.

The “Gorilla Banana” problem occurs when you need a method from an object, so you inherit from that object. it’s recommended to make classes “skinny” don’t have too many properties and methods.


Modular JavaScript:


It’s a good practice to separate our JS code in different files, but gather them by same or similar functions in order to have a good control, and project growing.

The coupling of code refers to how dependent certain elements or modules of code are on each other.

Node.js had already implemented modules before they were introduced in ES6, and used a slightly different notation called Common JS modules.
